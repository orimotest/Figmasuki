# Demo Flow

## Demo Mode の流れ

1. 要件タブで「セミナーサンプル」または「noteサンプル」を読み込む
2. 要件、ターゲット、訴求、トーンを確認する
3. 「この要件で自動制作へ」を押して制作ブリーフを作る
4. 制作タブで「自動制作を開始」
5. Figmaキャンバス上に Project Header、30 Ideas、15 Typography Drafts、5 Refined SVGs、Compare Result、Background Variations、Final Candidate が順番に記録される
6. 評価タブで、生成済み5案またはFigmaで選択した2〜5案を比較する
7. 背景生成ブリーフを確認し、必要に応じて背景案を生成してFigmaへ記録する

## セミナーデモで見る判断材料

- 入口として強い課題共感型と、内容理解に向く参加メリット型を比較できる
- 30案から15案、15案から5案へ絞る理由がボードに残る
- 背景案は文字領域を邪魔しないことを前提に確認する
- Final Candidate は背景レイヤーとSVG/テキストレイヤーを分けて、後編集しやすい前提で見る

## デモで見せるポイント

- UI内で現在工程と次に起きることが分かる
- Figma上に工程別ボードが残る
- SVG候補だけでなく、選定理由、懸念、背景方針もレビューできる
- 完成案だけではなく、生成過程と判断理由を説明できる
